Ecross-Qualtrics - How To Use

  1. Overview

    This program allows you to use the popular e-cross paradigm from psychology in Qualtrics. It keeps track of 6 important variables the number of "e"s in the text that you provide, the number of high depletion "e"s in the text you provide (high depletion is defined as any upper or lowercase "e" that not within two characters of any vowel within a given word), the number of letters that an individual clicked, the number of "e"s that an individual actually clicks on, and the number of high depletion "e"s that an individual clicked on. I also include an exact copy of the paragraph after the individual is done with the task, this is for your records. As an added bonus, it is impossible for a user to cheat. They can not use Ctrl+F to "Find" the "e"s in the paragraph you provide, nor can they go to their browser's menu and use the "Find" function that way.

    • The tricky part is that you are going to have to change the names in the "Embedded Data" block in the survey flow AND those names must correspond to the names in quotations in the JavaScript snippet. This is because if you implement the ecross paradigm properly you need a practice round, a low depletion round, and a high depletion round. This means that in your survey flow you are going to have this JavaScript snippet 3 times. Each of the three instances of the JavaScript snippet will need 3 corresponding "Embedded Data" blocks. I will refer to the "Embedded Data" block and JavaScript snippet as pairs. IF each of your pairs do not uniquely name the variables, then you will not get the right data. For instance if two of you pairs each have the same names for all 6 variables (as labeled in the JavaScript snippet and in the "Embedded Data" block) then you will get two copies of whichever block executed second, because the second block will overwrite the variable names.   7. Specifications Computerized E-Cross Task

    • Participants must be able to click on a letter to “cross” it out, meaning a small x or slash should appear through the letter. The program needs to prevent participants from using control+F to find the es.
    • Participants run through a shorter practice round with one paragraph, then the longer manipulation round with a different paragraph. Need to be able to control how long participants are given to complete each paragraph.
    • Must have data on:
      • Number of es successfully crossed out in practice round and low depletion
      • Number of es in the paragraph
      • Number of high depeltion es in the paragraph
      • Number of other letters crossed out
      • Number of correct es in high depletion round crossed out
      • A snapshot of the paragraph after a user has completed the task
